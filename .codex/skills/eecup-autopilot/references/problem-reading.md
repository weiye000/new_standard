# 读题行为规范

## 核心原则

读题阶段只做理解和拆解，不直接建模、不写主模型代码、不把猜测当成题面要求。

必须区分：

- 题面明示内容；
- 数据文件能够证明的事实；
- Codex 的合理推断；
- 需要用户确认的歧义。

## 输入材料检查

读题前先确认：

- 官方题面文件路径；
- 附件数据路径；
- 是否有论文模板或格式说明；
- 是否有补充说明、数据字典、附件说明；
- 文件能否正常读取，遇到pdf或者word调用相关skill去解决，务必读到所有信息。

缺官方题面或关键数据时，停止并只索要缺失文件。

## 逐题拆解步骤

对每个小问都必须填写：

| 项目 | 要求 |
|---|---|
| 原题要求 | 用简短原意复述，不照抄长段题面 |
| 输入 | 题面或数据给了什么 |
| 输出 | 最终要交付什么数字、表、图、方案或结论 |
| 约束 | 明示约束、工程约束、边界条件 |
| 评价指标 | 题面给定指标或需要构造的指标 |
| 单位和尺度 | 时间尺度、空间尺度、功率/电量/成本等单位 |
| 数据需求 | 需要哪些字段、哪些字段缺失或需构造 |
| 可能误读点 | 容易理解错的词、范围、目标或隐含条件 |
| 暂定理解 | Codex 的解释，必须标注为暂定 |
| 需确认问题 | 会影响模型路线的高影响问题 |

## 禁止行为

读题阶段禁止：

- 直接选择主模型；
- 直接大规模训练或优化；
- 把题面没有说的目标函数写成确定事实；
- 忽略单位和数据粒度；
- 只输出“可以用某某模型”的泛泛建议；
- 为了推进而跳过歧义。

允许做：

- 读取和摘要题面；
- 检查数据字段；
- 统计数据规模、缺失值和基本单位；
- 写基准模型候选草案；
- 列出模型路线需要的调研问题。

## 必须产出

读题阶段结束前创建或更新：

```text
00_problem/problem_decomposition.md
04_paper/paper_draft.md
06_management/PROJECT_STATE.md
06_management/HANDOFF.md
06_management/PHASE_REVIEW.md
06_management/risk_list.md
```

`problem_decomposition.md` 必须包含：

- 题目总体理解；
- 每个小问的拆解表；
- 数据字段和单位初表；
- 关键假设草案；
- 可能误读点；
- 需要用户确认的问题；
- 下一阶段调研关键词。

## 暂停规则

完成读题拆解后必须暂停。

暂停汇报时必须给用户：

- 一句话题目总体理解；
- 每个小问的目标摘要；
- 当前最大歧义；
- 是否可以进入公开资料调研；
- 若存在高影响歧义，列出可选解释及各自后果。
