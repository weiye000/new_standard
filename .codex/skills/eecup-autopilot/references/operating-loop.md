# 阶段运行规则

## 工作块

每个工作块只完成一个明确交付物：

1. 读取启动必读文件。
2. 判断当前阶段。
3. 执行一个交付物。
4. 验证输出。
5. 同步更新 `paper_draft.md`。
6. 更新 `PROJECT_STATE.md` 和 `HANDOFF.md`。
7. 若到达阶段关卡，更新 `PHASE_REVIEW.md` 并暂停。

## 阶段关卡

| 关卡 | 必须说明 |
|---|---|
| 题意拆解完成 | 任务、输入、输出、约束、评价指标、歧义和风险 |
| 调研选型完成 | 每个模型方案的优点、缺点、风险、实现难度、验证方式、论文价值 |
| 基准模型完成 | 基准结果、指标、能否支撑后续主模型、当前问题 |
| 模型异常 | 异常现象、可能根因、恢复方案优劣、推荐选择 |
| 验证分析完成 | 稳健性证据、敏感因素、结论可信度 |
| 论文整理完成 | 已完成章节、缺失结果、图表和支撑材料状态 |

## 停止条件

以下情况必须停止并询问：

- 官方题面或数据缺失；
- 题目存在两种互斥且都合理的解释；
- 需要用户选择模型路线或异常恢复方案；
- 大阶段结束，需要确认是否继续；
- 需要凭据、最终提交、联网下载、身份信息、破坏性或覆盖性操作。

## 每个小问的交付物

每个小问创建或更新：

- `00_problem/subproblem_X.md`
- `02_src/` 中的代码入口
- `03_results/tables/` 中的结果表
- 必要时在 `03_results/figures/` 中保存图
- `04_paper/paper_draft.md` 中对应正文片段
