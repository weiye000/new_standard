# 阶段运行规则

## 工作块

每个工作块只完成一个明确交付物：

1. 读取启动必读文件。
2. 检查当前目录是否为 Git 仓库；若不是，先初始化仓库并说明。
3. 判断当前阶段。
4. 执行一个交付物。
5. 验证输出。
6. 同步更新 `paper_draft.md`。
7. 更新 `PROJECT_STATE.md` 和 `HANDOFF.md`。
8. 检查 `git status --short`；若有重要进度，创建本地提交，提交信息使用清晰前缀。
9. 若到达阶段关卡，更新 `PHASE_REVIEW.md`，必要时再次提交阶段状态，然后暂停。

## Git 保存点

以下情况视为重要进度，必须形成本地提交：

- 题意拆解、调研记录或模型候选方案发生实质变化；
- 数据处理脚本、模型代码、实验配置或随机种子发生变化；
- 结果表、图、指标、实验日志或论文片段发生变化；
- 项目规则、skill、支撑材料或交接文件发生变化。

远程推送不自动执行。需要 push 时，先说明 commit hash、变更概要、目标远程和目标分支，等待用户确认。

## 阶段关卡

| 关卡 | 必须说明 |
|---|---|
| 题意拆解完成 | 任务、输入、输出、约束、评价指标、歧义和风险 |
| 调研选型完成 | 每个模型方案的优点、缺点、风险、实现难度、验证方式、论文价值 |
| 基准模型完成 | 基准结果、指标、能否支撑后续主模型、当前问题 |
| 模型异常 | 异常现象、可能根因、恢复方案优劣、推荐选择 |
| 验证分析完成 | 稳健性证据、敏感因素、结论可信度 |
| 论文整理完成 | 已完成章节、缺失结果、图表和支撑材料状态 |

## 停止条件

以下情况必须停止并询问：

- 官方题面或数据缺失；
- 题目存在两种互斥且都合理的解释；
- 需要用户选择模型路线或异常恢复方案；
- 大阶段结束，需要确认是否继续；
- 需要凭据、最终提交、联网下载、身份信息、远程推送、破坏性或覆盖性操作。

## 每个小问的交付物

每个小问创建或更新：

- `00_problem/subproblem_X.md`
- `02_src/` 中的代码入口
- `03_results/tables/` 中的结果表
- 必要时在 `03_results/figures/` 中保存图
- `04_paper/paper_draft.md` 中对应正文片段
